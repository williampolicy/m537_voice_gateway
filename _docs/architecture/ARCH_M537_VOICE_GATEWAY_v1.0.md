## 1. 架构概览

### 1.1 系统定位

M537 Voice Gateway 是 LIGHT HOPE 生态系统的语音交互入口，提供自然语言查询能力。

## 2. 分层架构

### 2.1 表现层 (Presentation Layer)

| 组件 | 职责 |
|------|------|
| REST API | HTTP 请求处理 |
| WebSocket | 实时双向通信 |
| 静态文件服务 | 前端资源托管 |

### 2.2 应用层 (Application Layer)

| 组件 | 职责 |
|------|------|
| 路由 (Routes) | 请求路由与参数验证 |
| 中间件 (Middleware) | 横切关注点处理 |
| 控制器逻辑 | 业务流程编排 |

### 2.3 领域层 (Domain Layer)

| 组件 | 职责 |
|------|------|
| Intent Parser | 意图识别 |
| Query Executor | 查询执行 |
| Response Builder | 响应构建 |
| Tool Registry | 工具管理 |

### 2.4 基础设施层 (Infrastructure Layer)

| 组件 | 职责 |
|------|------|
| Cache | 查询缓存 |
| Logging | 日志管理 |
| Tracing | 分布式追踪 |
| Metrics | 指标采集 |

## 3. 核心模块设计

### 3.1 意图识别模块

```
输入: "现在有多少个项目"
         │
         ▼
┌─────────────────────────┐
│    Pattern Matching     │
│  (正则表达式匹配)        │
└────────────┬────────────┘
             │
             ▼

│  (置信度计算)            │

             │
             ▼

```

### 3.2 查询执行模块

```
输入: (intent, params)
         │
         ▼
┌─────────────────────────┐
│    Cache Check          │──── 命中 ───▶ 返回缓存
│  (缓存检查)              │
└────────────┬────────────┘
             │ 未命中
             ▼
┌─────────────────────────┐
│    Tool Lookup          │
│  (工具查找)              │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│    Tool Execution       │
│  (工具执行)              │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│    Cache Store          │
│  (缓存存储)              │
└────────────┬────────────┘
             │
             ▼
输出: {success: true, data: {...}}
```

### 3.3 缓存策略

| 意图类型 | TTL | 理由 |
|----------|-----|------|
| count_projects | 5min | 静态数据 |
| system_status | 30s | 动态数据 |
| list_ports | 1min | 半动态 |
| service_logs | 15s | 高动态 |

## 4. 中间件栈

### 4.1 中间件顺序 (由外到内)

```
请求 ──▶ SecurityHeaders
             │
             ▼
         GZip
             │
             ▼
         CORS
             │
             ▼
         InputSanitization
             │
             ▼
         RateLimit
             │
             ▼
         Authentication
             │
             ▼
         Tracing
             │
             ▼
         ProcessTime
             │
             ▼
        [Application]
```

### 4.2 中间件职责

| 中间件 | 职责 |
|--------|------|
| SecurityHeaders | 添加安全响应头 |
| GZip | 响应压缩 (>500 bytes) |
| CORS | 跨域资源共享 |
| InputSanitization | 输入清理 |
| RateLimit | 请求限流 |
| Authentication | API Key 验证 |
| Tracing | 分布式追踪 |
| ProcessTime | 处理时间记录 |

## 5. 弹性设计

### 5.1 熔断器状态机

```
     ┌──────────────────────────────────────────┐
     │                                          │
     ▼                                          │
┌─────────┐    失败达阈值    ┌─────────┐       │
│ CLOSED  │ ──────────────▶ │  OPEN   │       │
│ (正常)   │                 │ (熔断)   │       │
└────┬────┘                 └────┬────┘       │
     │                           │             │
     │                      超时后尝试          │
     │                           │             │
     │                           ▼             │
     │                    ┌───────────┐        │
     │      成功恢复      │ HALF_OPEN │        │
     └──────────────────── │ (半开)    │────────┘
                          └───────────┘   失败
```

### 5.2 优雅关闭流程

```
收到 SIGTERM
     │
     ▼
┌─────────────────────────┐
│ 1. 停止接受新请求       │
│    (返回 503)            │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│ 2. 等待现有请求完成      │
│    (drain_timeout: 10s)  │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│ 3. 执行清理回调         │
│    (关闭连接/保存状态)   │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│ 4. 退出进程             │
└─────────────────────────┘
```

## 7. 安全架构

### 7.1 安全边界

```
┌─────────────────────────────────────────────────────────────┐
│                        外部网络                             │
└──────────────────────────┬──────────────────────────────────┘
                           │
                    ┌──────▼──────┐
                    │   防火墙     │
                    └──────┬──────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                        DMZ                                   │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │                    Nginx/LB                              │ │
│  │  • SSL 终止                                              │ │
│  │  • DDoS 防护                                             │ │
│  │  • WAF 规则                                              │ │
│  └────────────────────────┬────────────────────────────────┘ │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                      内部网络                                │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │                 M537 Voice Gateway                       │ │
│  │  • API Key 认证                                          │ │
│  │  • 速率限制                                              │ │
│  │  • 输入验证                                              │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 认证流程

```
请求 + API Key
     │
     ▼
┌─────────────────────────┐
│  提取 API Key           │
│  (Header/Query)         │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  验证 Key               │ ──── 无效 ───▶ 401
│  (Hash 比对)            │
└────────────┬────────────┘
             │ 有效
             ▼
┌─────────────────────────┐
│  检查限流               │ ──── 超限 ───▶ 429
│  (令牌桶)               │
└────────────┬────────────┘
             │ 通过
             ▼
        继续处理
```
