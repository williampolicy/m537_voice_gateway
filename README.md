## 核心价值

**构建 LIGHT HOPE 生态的统一语音访问层** —— 让每个用户都能通过自然语言访问属于自己的数据、任务、状态和建议。

这不是"做一个语音功能"，而是**平台级基础设施**。

| 用户类型 | 需求 | 价值 |
|---------|------|------|
| 运维人员 | 语音查询服务器状态 | 解放双手，快速获取信息 |
| 开发者 | 快速了解项目状态 | 碎片时间高效利用 |
| 管理者 | 系统健康概览 | 随时随地掌握全局 |

## 快速开始

```bash
# 1. 进入项目目录
cd /data/projects/m537_voice_gateway

# 2. 复制环境变量
cp .env.example .env

# 3. 启动服务
docker compose up -d --build

# 4. 访问
# 前端: http://localhost:5537
# 健康检查: http://localhost:5537/health
# API 文档: http://localhost:5537/docs
```

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/voice-query | 语音查询主接口 |
| GET | /api/health | 健康检查 (V5.3) |
| GET | /api/metrics | Prometheus 指标 |
| GET | /api/version | 版本信息 |

### 请求示例

```bash
curl -X POST http://localhost:5537/api/voice-query \
  -H "Content-Type: application/json" \
  -d '{"transcript": "现在有多少个项目"}'
```

### 响应示例

```json
{
  "success": true,
  "data": {
    "answer_text": "当前共有 327 个项目",
    "intent": "count_projects",
    "confidence": 0.95
  }
}
```

## 安全边界

**绝对禁止**:
- LLM 直接执行 shell 命令
- 前端直接访问服务器内部
- 未经白名单的任何操作
- 访问敏感路径 (/etc, /root, ~/.ssh)

**允许**:
- 只读查询
- 白名单脚本调用
- 结构化数据返回
